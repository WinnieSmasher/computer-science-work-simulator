# 27 关路线图

这份路线图重新按 PDF 中的 27 个“模拟工作”能力点对齐。仓库里的 level-XX.md 只是文件命名；课程语义上对应的是“第 XX 关”。

目标不是做 27 个玩具 demo，而是把后端开发新人在真实团队里会遇到的工作拆成 27 次训练：理解需求、模仿团队规范、写接口、排查 bug、处理脏数据、对接第三方、中间件、事务、部署、异步、微服务、复杂 C 端方案和技术文档。

## 设计原则

- 每一关都必须能落到后端开发工作：接口、表、服务、事务、缓存、MQ、部署、日志、文档或方案评审。
- 业务场景可以变化，但都放在 NovaMart 这个业务生态里，避免变成互不相关的小玩具。
- 不复制 PDF 原文和营销话术，只保留能力结构，并转写成可执行的工程训练。
- 后续代码实现按关卡逐步提交，每一关都要有测试、文档或 checklist。
