# Roadmap

NovaMart 的 27 关不是 27 个孤立 demo，而是一个系统从简单到复杂的演进。

| Level | 业务任务 | 工作能力 | v1 状态 |
| --- | --- | --- | --- |
| 01 | 画出 NovaMart 用户购物主流程和后台商品管理流程 | 理解产品、业务流程、角色边界 | 文档 |
| 02 | 按已有接口风格新增商品标签 API | 模仿团队接口/数据库/返回结构规范 | 代码+测试 |
| 03 | 实现商品 CRUD、分页、搜索、逻辑删除 | 理解简单技术需求并落地 | 代码+测试 |
| 04 | 用 IDE/debug/日志定位商品搜索 bug | 快速掌握工具、看报错、调试 | 文档 |
| 05 | 实现用户端商品列表 `hasMore/isEnd` 分页 | 理解 C 端列表、状态、时间与分页逻辑 | 文档 |
| 06 | 给商品评论接口加参数校验、SQL 注入防护、统一异常 | 跨端安全协同 | 文档 |
| 07 | 引入代码规范、DTO/VO、MyBatis Plus、模板化代码 | 代码规范和描述语言 | 文档 |
| 08 | 处理历史脏商品数据、mock 库存接口、兼容旧字段 | 数据污染和 mock | 文档 |
| 09 | 对接模拟 OSS 文件上传服务 | 阅读文档并对接第三方接口 | 文档 |
| 10 | 用 EasyExcel 做商品批量导入导出 | 借助 AI/文档快速学习新技术 | 文档 |
| 11 | 梳理一次请求从网关到 Controller/Service/DB 的链路 | 请求生命周期、线程池、连接池 | 文档 |
| 12 | 实现登录注册、JWT、cookie/session 对比实验 | 前后端互相理解、认证鉴权 | 文档 |
| 13 | 在已有订单模块上新增优惠券能力 | 在他人代码上迅速开发 | 文档 |
| 14 | 实现下单事务：订单、库存、优惠券、支付记录 | 靠逻辑写稍复杂业务代码 | 文档 |
| 15 | Docker 部署单体服务，Nginx 反代，查看日志 | 基本运维和线上实操 | 文档 |
| 16 | 加入 GitHub Actions、Docker Compose、环境变量配置 | 基础设施工具和 CI/CD | 文档 |
| 17 | 下单成功后异步发送通知 | 简单异步架构设计 | 文档 |
| 18 | Redis 缓存商品详情、库存热点数据 | 快速熟悉 Redis 中间件 | 文档 |
| 19 | 设计读写分离和缓存一致性方案文档 | 理解不完美与工程取舍 | 文档 |
| 20 | 将商品、订单、用户边界拆成模块/服务 | 理解服务颗粒度并按图落地 | 文档 |
| 21 | 用 RabbitMQ 实现订单事件、库存扣减、通知消费 | 快速熟悉 MQ 中间件 | 文档 |
| 22 | 实现复杂 C 端推荐/瀑布流/曝光去重需求 | 理解复杂用户端业务 | 文档 |
| 23 | 重构一个过度设计模块，移除不必要 MQ/锁/抽象 | 理解过度设计是魔鬼 | 文档 |
| 24 | 为订单履约写技术方案、接口文档、流程图 | 书写技术文档 | 文档 |
| 25 | 增加前端管理台最小页面或 API client | 根据经验快速理解新技术 1 | 文档 |
| 26 | 设计分库分表、Canal、Seata 的可选演进方案 | 根据经验快速理解新技术 2 | 文档 |
| 27 | 阅读并解释完整“秒杀/库存/支付/通知”复杂方案 | 理解复杂技术解决方案 | 文档 |
