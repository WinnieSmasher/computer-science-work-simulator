# 后端代码自查清单

每完成一关，都先按这份清单自己扫一遍，再提交。

## 接口契约

- 成功响应是否都使用 `ApiResponse.ok(...)`。
- 失败响应是否仍然使用统一 envelope。
- 错误码是否能让前端区分校验失败、业务失败和请求格式失败。
- 分页接口是否返回 `page/size/total/hasMore/isEnd`。

## 参数校验

- 前端传来的字段是否全部重新校验。
- 枚举、数字、日期、列表元素是否有边界测试。
- DTO 构造器是否会抢在 Bean Validation 之前抛异常。
- 错误信息是否避免泄露数据库字段、堆栈、token、cookie。

## 业务边界

- 用户端接口是否过滤草稿、下架、逻辑删除数据。
- 后台排查接口是否能解释“数据存在但前台不可见”。
- 逻辑删除、重复提交、重复消息是否有清晰行为。
- 事务边界是否覆盖了必须同时成功或失败的写操作。

## 测试

- 测试是否验证行为，而不是只验证实现细节。
- 是否覆盖正常路径、异常路径和边界值。
- 测试数据是否通过 keyword、返回 id 或 fixture 隔离，避免依赖执行顺序。
- 能不用重启 Spring 上下文就不用重启。

## 文档

- API 文档是否同步更新。
- 关卡验收标准是否能被别人独立执行。
- 技术方案是否写出“不做什么”和“为什么暂时不做”。
- 引入新技术时是否写清楚成本、收益和替代方案。
