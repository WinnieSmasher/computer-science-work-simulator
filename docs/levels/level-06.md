# 第 06 关 - 跨端安全协同

## 对应能力

- 能力简称：跨端安全协同
- 能力说明：前端不可信，后端要校验、防注入、统一异常、权限控制

## 后端模拟工作场景

前端提交商品评论和昵称，后端需要判断哪些字段可信，哪些必须重新校验。

## 这一关真正练什么

前端不可信，后端要校验、防注入、统一异常、权限控制

## 相关技术关键词

SQL 注入、input、textarea、Jackson、JSON 库、Git 分支、测试、全局异常

## 你接到的任务

- 新增商品评论创建和查询接口。
- 对内容长度、HTML、空白字符、商品状态做后端校验。
- 统一异常返回，不把堆栈或数据库字段暴露给前端。

## 验收标准

- 恶意输入不会进入持久层。
- 前端传来的 userId、price、status 不被直接信任。
- 测试覆盖校验失败、商品不存在和正常评论。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
