# 第 16 关 - 基础设施工具

## 对应能力

- 能力简称：基础设施工具
- 能力说明：Docker、GitLab、CI/CD、负载均衡、分布式基本概念

## 后端模拟工作场景

团队要求本地环境和 CI 标准化，新人不能再靠“我电脑能跑”。

## 这一关真正练什么

Docker、GitLab、CI/CD、负载均衡、分布式基本概念

## 相关技术关键词

集群、分布式、LB、Docker、GitLab、Codeup、CI/CD

## 你接到的任务

- 完善 MySQL、Redis、RabbitMQ 的 Docker Compose。
- CI 中运行编译和单元测试。
- 把配置改成环境变量或 profile 可切换。

## 验收标准

- 新同学按 README 能启动基础设施。
- CI 失败能阻止合并。
- 能说明 Docker、CI、LB 分别解决什么问题。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
