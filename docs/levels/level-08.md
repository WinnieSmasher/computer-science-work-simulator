# 第 08 关 - 数据污染/mock

## 对应能力

- 能力简称：数据污染/mock
- 能力说明：处理脏数据、测试数据、历史数据兼容、mock 接口

## 后端模拟工作场景

历史商品数据里有脏状态、空价格和失效标签，同时库存服务还没开发完。

## 这一关真正练什么

处理脏数据、测试数据、历史数据兼容、mock 接口

## 相关技术关键词

不用外键的原因、联表查询、脏数据、Mock、${} 使用场景、拆联表

## 你接到的任务

- 编写商品数据质量检查脚本或接口。
- 为库存查询提供 mock adapter。
- 兼容旧字段并记录迁移计划。

## 验收标准

- 脏数据会被报告而不是让接口随机失败。
- mock 库存能被测试替换。
- 能解释为什么线上系统常常不用强外键。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
