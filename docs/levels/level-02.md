# 第 02 关 - 培养模仿设计的能力

## 对应 PDF 能力

培养模仿设计的能力

## 后端模拟工作场景

团队已有商品标签接口、返回结构和文档风格，现在要新增“运营活动分组”接口。

## 这一关真正练什么

模仿既有接口、数据库和文档规范，在团队风格内开发新能力。

## 相关技术关键词

接口文档四要素、REST 工程取舍、JSON 类型约定、数据库文档、索引、存储引擎

## 你接到的任务

- 阅读现有标签 API 的命名、返回结构、错误码和测试风格。
- 设计活动分组的创建、列表、详情、更新、禁用接口。
- 补充数据库字段说明、索引说明和接口示例。

## 验收标准

- 新增接口风格与已有代码一致。
- 文档包含路径、请求、响应、错误码四要素。
- 能说明为什么不盲目追求“纯 REST”。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
