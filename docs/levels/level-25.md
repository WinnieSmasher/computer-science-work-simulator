## 后端模拟工作场景

新团队使用 Vue 管理台、Gateway、Zipkin 和 ELK，你主要做后端，但必须能理解调用链和日志。

## 这一关真正练什么

基于已有 Web/API 经验快速接手新技术栈周边。

## 相关技术关键词

Vue、Element UI、axios、ES、Dubbo、Zipkin、Gateway、logback、ELK

## 你接到的任务

- 提供一个最小后台 API client 或页面调用商品接口。
- 让后端日志带 traceId，并说明如何进入 ELK 检索。
- 画出 Gateway 到服务再到日志系统的链路。

## 验收标准

- 能用最小页面或 API client 完成商品创建。
- 错误码能从后端传到前端展示。
- 能解释 Gateway、Zipkin、ELK 各自解决什么问题。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
