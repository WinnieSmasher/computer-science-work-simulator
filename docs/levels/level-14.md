# 第 14 关 - 靠逻辑写稍复杂代码的能力

## 对应 PDF 能力

靠逻辑写稍复杂代码的能力

## 后端模拟工作场景

下单要同时创建订单、扣库存、占用优惠券、创建支付记录，多个模块互相依赖。

## 这一关真正练什么

靠业务逻辑和事务边界完成稍复杂后端代码。

## 相关技术关键词

事务、ACID、线上 MySQL 第一行代码、循环依赖、MVCC、Spring 声明式/编程式事务

## 你接到的任务

- 实现下单事务主流程。
- 用领域服务打破循环依赖，而不是互相注入一圈。
- 处理库存不足、优惠券失败、支付记录创建失败的回滚。

## 验收标准

- 关键写操作在清晰事务边界内。
- 失败时不会出现半张订单。
- 能解释 MVCC 和事务隔离对这个需求的影响。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
