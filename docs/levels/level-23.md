# 第 23 关 - 理解过度设计是魔鬼的能力

## 对应 PDF 能力

理解过度设计是魔鬼的能力

## 后端模拟工作场景

前任同学给普通库存预占做了 MQ、MongoDB、分布式锁、补偿任务和多层抽象，维护成本爆炸。

## 这一关真正练什么

识别并移除过度设计，保留当前规模真正需要的方案。

## 相关技术关键词

代码中的引擎、MQ、MongoDB、乐观锁、分布式锁、补偿、Quartz、xxl-job

## 你接到的任务

- 画出现有过度设计链路。
- 保留必要事务和幂等，移除不必要抽象。
- 写出什么时候才需要恢复 MQ/分布式锁。

## 验收标准

- 重构后测试仍通过。
- 代码路径更短，职责更清楚。
- 文档说明“为什么不做”而不是只写“没必要”。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
