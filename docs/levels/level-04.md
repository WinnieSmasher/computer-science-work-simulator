# 第 04 关 - 通过 AI 快速掌握简单工具的能力

## 对应 PDF 能力

通过 AI 快速掌握简单工具的能力

## 后端模拟工作场景

运营反馈商品搜索有时搜不到，导师只给你日志片段和一个失败请求。

## 这一关真正练什么

使用 IDE、debug、日志和 AI 辅助快速定位问题，而不是重写整套搜索。

## 相关技术关键词

IDEA、debug、抓包、看报错、动态 SQL、MyBatis 兜底、日志

## 你接到的任务

- 先写失败用例复现 keyword 大小写、空格或状态过滤问题。
- 用断点跟踪 Controller、Service、Repository。
- 补充不泄露敏感信息的 debug 日志。

## 验收标准

- 能复现问题并写出根因。
- 修复后保留回归测试。
- 能说明日志里哪些信息不能打印。

## 学习提醒

这一关不是为了堆技术名词，而是训练“接到真实后端工作后如何理解、拆解、实现、验证、解释”的能力。代码能跑只是底线，能说明为什么这样做才算过关。
